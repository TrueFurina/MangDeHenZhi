## 2. 产品定义

**Product Goals（3 个，正交）**
- **G1 闭环可演示**：20 天内交付"登录→测评→报告→课程/路径→证书(模拟链)"完整闭环，评委一眼看懂"学-练-证"价值。
- **G2 复用零成本**：仅使用已实现且验证过的模块（Auth / Assessment / Course / Certification / Metaverse 原型），不做新架构、不接真实链。
- **G3 措辞一致**：演示稿/申报书表述与实现严格对齐，不夸大未兑现能力（区块链/元宇宙互动/自适应路径）。

**User Stories（4 条）**
1. 作为**评委/观众**，我想看到学员从登录到拿到证书的完整流程，以便理解产品核心价值。
2. 作为**学员**，我想完成一次 AI 测评并看到能力雷达图与推荐课程，以便获得个性化反馈。
3. 作为**演示者**，我想一键进入 3D 元宇宙场景看静态展示，以便直观感受"实训"概念。
4. 作为**学员**，我想通过测评后自动获得一张带哈希/模拟上链信息的证书，以便理解"可验证凭证"叙事。

## 3. 技术规范 — Requirements Pool（含验收标准）

图例：🔴 P0 必须（演示必备）  🟡 P1 应有（演示增强）  ⚪ P2 可选（可砍）

### 🔴 P0 — 最小可演示闭环（必做）

| ID | 功能 | 验收标准（可度量） |
|----|------|------------------|
| F1 | 用户登录/注册（JWT） | 用测试账号 admin/teacher/student 均可登录；token 写入 localStorage；未登录访问 `/dashboard` 等受保护路由跳 `/login`；路由守卫生效。 |
| F2 | AI 能力测评（12 题 / 3 维） | 进入 `/assessments` → 选测评 → 逐题作答（进度条+圆点导航）→ 提交；提交后生成 `AssessmentResult`；12 题流程无阻断。 |
| F3 | AI 测评报告（DeepSeek） | 结果页（`/assessments/results/:id`）展示总体评价/优势/薄弱/建议 + 雷达图；**配置 `DEEPSEEK_API_KEY` 时返回真实报告；未配置时降级本地模拟且不报错**。 |
| F4 | 课程浏览/详情/分类 | `/courses` 列表 + 分类筛选可用；`/courses/:id` 详情（元数据/前置要求/相关推荐）可打开。 |
| F5 | 学习路径/课程推荐 | 结果页展示 AI 推荐课程 + 路径步骤；基于最近一次测评结果返回，**空结果有兜底**（不白屏）。 |
| F6 | 证书签发(模拟链) + 我的证书 + 哈希验证 | 测评**通过**后 `CertificationService` 自动签发证书（DB）；`/certifications` 展示证书卡（哈希/模拟 txId/区块链信息，且**明确标注"模拟上链"**）；`/certifications/verify/:hash` 返回 `verified=true`。 |

## 4. Lite 明确砍掉清单（范围外，不在演示闭环）

| 砍掉项 | 原因 | Lite 替代 |
|--------|------|----------|
| **区块链真实上链（Fabric 网络）** | 无真实链、备案周期长 | `BlockchainService` 模拟 txId，证书标注"模拟上链" |
| **元宇宙 AI 角色实时互动（WebRTC/对话后端）** | 无实时后端，原型仅静态 | 静态 3D 场景 + 光环标签 |
| **自适应学习路径引擎 / 督学 / 学习追踪** | 无追踪与动态调整能力 | 现有 `RecommendationService` 文本建议 |
| 搜索 / 管理后台 / PWA / BI | 非演示核心闭环 | P2 或留真版 |
| 雇主生态 / HR 对接 / 学分银行 | 全部超出 Lite | 真版 |
| 埋点（MCL / AARRR） | 度量地基，非演示必需 | 真版优先 |

> **演示剧本建议（5 分钟）**：登录 student → 进入测评 → 12 题作答 → 看 AI 报告+雷达图+推荐课程 → 点进课程详情 → 回"我的证书"看自动签发证书（标注模拟上链）→ 切元宇宙静态场景展示 → 收尾讲真版蓝图（真实上链+实时互动+自适应）。

## 6. 真版 PRD 大纲（供 software-architect 接手）

**标题**：芒得很职真版 PRD（大创中期检查 · 2026-11）
**范围**：补齐全 A（测评深化）/ B（元宇宙互动）/ C（真实上链）三支柱 + 度量与合规。
**与 Lite 的关系**：Lite 是"壳+闭环演示"；真版在壳上替换模拟件为真实能力。

| # | 章节 | 关键内容 | 前置依赖 |
|---|------|---------|---------|
| 1 | 真实区块链上链 | Hyperledger Fabric 接入；**网信办备案前置**；OpenBadges/Blockcerts 兼容；HR/招聘可验证生态 | 备案（最长周期，Q3 即启动） |
| 2 | 元宇宙 AI 角色实时互动 | WebRTC/对话后端；虚拟面试标杆场景；性能/移动端轻量化 | B 立项（Q3 启动，Q4 标杆） |
| 3 | 自适应学习路径引擎 | 学习追踪 + 动态重规划 + 轻量督学；对齐 +15~20pp 完课率承诺 | A 技术债治理 |
| 4 | 埋点体系 | 北极星 MCL + AARRR P0/P1/P2；**先上 P0：测评完成/闭环完成** | 作为一切度量的地基 |
| 5 | 内容与题库扩充 | 标杆自研 + 聚合（参考 Degreed）；职业/学科覆盖 | 内容策略 |
| 6 | 企业版 / BI | 技能数据 → 业务结果可视化 | B/C 数据贯通 |
| 7 | 合规与安全 | 区块链备案、个人信息保护设计 | 法务 + 研发并行 |
| 8 | 里程碑 | Q3 备案+立项 → Q4 B 标杆 + C MVP → 2027 生态打通 | 依赖链倒挂（备案→MVP） |

**关键风险（真版）**：① 备案为最长前置硬依赖，倒挂 Q4 MVP；② B 原型成熟度低（372 行）；③ A 仍欠技术债；④ 三层同铺易稀释资源 —— 优先级聚焦 A 基本盘 + B/C 卡位。
