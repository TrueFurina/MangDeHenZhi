## 2. 用户反馈摘要（来自瑞思）

**三类核心人群及第一痛点**

| 人群 | 占比/特征 | 第一痛点 |
|------|-----------|----------|
| 求职学员 | — | 岗位能力不匹配 / 简历无凭证 |
| 转行在职者 | >70% | 学不完 / 没时间 / 要个性化 |
| 企业 HR | — | 招不到可验证技能的人 |

**跨人群第一痛点** =「学得完 + 学得准」（完课率 + 个性化路径）。

**路线图排期建议（用户视角）**：P0 = AI 测评 + 个性化学习路径 + 轻量督学（Q3 立即）；P1 = Web 轻量 3D 实训企业侧（Q4）；P2 = 区块链 + 雇主生态（远期）。

**用户价值 → 路线图特性映射**：学得准 → AI 测评 + 能力画像；学得完 → 个性化路径 + 督学；凭证 → 区块链证书；招聘对接 → HR 生态。

## 3. 竞品动态（来自竞析）

**竞品已发生动作（已 WebSearch 核验）**：云学堂→绚星「AI First」战略；三节课「AI 认证地图」；Coursera 中国本地化 + 就业直通车（联合腾讯云/华为）；巨头（腾讯/阿里/字节/百度）拼 AI 超级入口；元宇宙政策转刚需（教育部 2026.2 加强虚拟仿真实训基地、「十五五」到 2030 建 500 个国家级基地）。

**竞品预测（未来 6–12 月，推断为主）**：云学堂/三节课补 AI 完成但不碰 VR/区块链；三节课向微证书延伸（威胁 C 叙事但非可验证凭证）；Coursera 维持 2D 不进 VR/区块链；巨头以 BaaS/教育云间接下场（蚂蚁链/腾讯云 TBaaS 可能开放区块链证书能力）。

**巨头下场时间窗（推断）**：自建垂直 A+B+C 概率低且慢，18–24 个月内不会正面竞争整合叙事；但 BaaS 能力开放可能 6–12 个月发生。结论：芒德有 **12–18 个月「整合叙事」窗口**；C 层护城河需在巨头 BaaS 开放前以「合规 + 品牌 + HR 生态」建立。

**政策红利（最强确定性，已核验）**：国务院《AI+行动意见》(2025.8)、《大规模职业技能提升培训行动》(2025.6，2025–2027 补贴性培训 3000 万人次)、《双千计划》、工信部元宇宙行动计划到 2026 产业 3500 亿；微证书/可验证凭证趋势（85% 雇主采用技能评估替代学历）。

## 5. 利益相关者沟通要点（按受众：高管/工程/设计 三版本）

### 【高管视图】— 战略 / 窗口期 / 投资回报 / 政策红利
1. **整合叙事独占窗口 12–18 个月**：芒德是唯一切面覆盖 A+B+C 的产品，竞品明确不进 VR/区块链，巨头自建垂直整合 18–24 月内不会正面竞争——时间窗稀缺，须在此内卡位。
2. **政策红利确定性最高、需优先承接**：《AI+行动意见》(2025.8)、2025–2027 补贴培训 3000 万人次、双千计划、元宇宙到 2026 产业 3500 亿、2030 建 500 国家级基地——B/C 双线均踩中政策刚需。
3. **投资回报锚点清晰**：认证溢价 2.3×、付费转化中位 2.83%；北极星 MCL 可直接关联营收与政策补贴入口。
4. **护城河打法**：C 层不与巨头 BaaS 硬拼技术，而是以「合规备案 + 品牌 + HR 生态」在 BaaS 开放前（6–12 月）建立先发壁垒。
5. **资源投放顺序**：Q3 即并行启动 B 立项 + C 备案（小成本卡位/合规），A 持续投入保基本盘；避免把 C 技术投入前置而拖垮备案时间线。

### 【工程视图】— 模块成熟度 / 技术债 / 排期依赖 / 埋点体系
1. **模块成熟度现状**：A=部分可用（DeepSeek 已集成，限流/健康检查已做，仍欠技术债）；B=部分可用（Three.js 约 372 行原型，需工程化）；C=预留未上线（零代码）。
2. **C 是最大排期风险点**：合规备案为长周期前置硬依赖，Q4 MVP 倒挂于备案；建议 Q3 即拉起合规+区块链并行流，避免年底集中阻塞。
3. **技术债治理优先级**：A 下一阶段聚焦能力画像与个性化路径算法；B 需从 372 行原型升级为企业级组件（性能/WebRTC 稳定性/移动端轻量化）。
4. **埋点体系须与路线图同步**：P0（测评完成、闭环完成）先于一切上线，作为 MCL 与 AARRR 度量地基；P1/P2 随模块上线补齐，避免「功能上了数不清」。
5. **跨模块数据贯通是 2027 重点**：技能信用闭环要求 A→B→C 数据流打通，架构上需提前规划统一身份/证书/能力画像数据模型，避免后期返工。

### 【设计视图】— 用户体验主线 / 完课率与个性化 / 证书可信感知
1. **用户体验主线 =「学得准 → 学得完 → 拿凭证」**：完美映射三大支柱（A 能力画像 / 路径+督学 / C 区块链证书），应作为统一 UX 叙事贯穿全产品。
2. **完课率是第一体验 KPI**：跨人群第一痛点「学得完 + 学得准」，行业完课率仅 30–50%，轻量督学可 +15~20pp；Q3 个性化路径 + 轻量督学是体验改版 Top1。
3. **个性化路径体验**：>70% 用户为转行在职者（没时间/要个性化），路径须「低投入高确定性」，用 AI 测评结果驱动动态推荐，减少无效学习。
4. **证书可信感知设计**：C 区块链证书核心价值在「可验证」，UX 上须让学员/HR 一眼感知不可篡改与权威背书（验证入口、链上凭证展示、HR 可查），对抗三节课「微证书」叙事。
5. **企业侧（HR）体验是 B/C 落地杠杆**：HR 最痛「招不到可验证技能的人」，B 实训标杆 + C 可验证凭证需设计 HR 视角招聘对接体验，形成 B→C→雇主生态闭环。
