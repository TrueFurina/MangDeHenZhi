### 【工程视图】— 模块成熟度 / 技术债 / 合规风险 / 埋点前置
1. **A 隐性 Bug 比技术债更急**：推荐维度键中英不匹配（前端 `communication` vs 后端「沟通能力」）→ 恒返 TECHNOLOGY；`getLearningPath` 控制器硬编码静态串、不调 service。**修复清单**：统一维度键（前端改中文 or 后端 map 加英文键）、让 `/learning-path` 调 `generateLearningPath`。
2. **C 是 100% mock，须当作「接线已有桩」而非「从 0」**：Q4 MVP = 接回 `CertificationService:40` 注释行 + `blockchainTxId` 持久化 + 真实 Fabric/OpenBadges；pom 需补 Fabric 依赖；`verifyOnChain` 硬编码 JSON 须替换为真实查询。
3. **over-claim 合规风险（P0）**：`Certifications.vue:8` 写死「全球可验证」，verify 仅翻 DB——**上线前必须按 `blockchainTxId` 是否存在条件渲染**，否则虚假宣传。
4. **埋点 P0 先于一切**：当前零事件体系，MCL/AARRR 不可度量；Sprint 1 第 0 天建事件表+采集管道，并同步埋「测评完成/闭环完成/实训完成」。

### 【设计视图】— 用户体验主线 / 完课率 / 证书可信感知
1. **UX 主线 =「学得准 → 学得完 → 拿凭证」**：A 能力画像 / 路径+督学 / C 区块链证书；但当前「学得准」因推荐 Bug 失效、「学得完」因督学缺失无抓手——须先补这两块体验才成立。
