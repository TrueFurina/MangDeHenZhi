# 概览：芒得很职用户研究计划与分析框架

**产出日期**：2026-07-17 ｜ **作者**：UX Researcher

## 做了什么
为芒得很职（AI+元宇宙+区块链 职业技能培训认证平台）制定了一份**混合方法用户研究计划**，并同步给出结果分析框架与一份**基于现有文档的二次研究综合（Phase 0）**。

## 关键内容
- **研究问题 RQ1–RQ6**：覆盖 C 端需求、痛点验证、三支柱价值排序、Lite 闭环可用性、B 端采购、无障碍。
- **方法论**：探索性序列混合设计（案头 → 定性访谈 → 定量问卷 → 可用性测试 → 三角验证）。
- **样本量统计依据**：访谈 n=12–15/段（主题饱和）、可用性 8–10（Nielsen）、问卷 C 端 ≥200 / B 端 ≥30（信效度 + 细分 ANOVA）。
- **结果分析框架**：主题分析六步法 + 痛点矩阵（严重度×普遍度）+ 信效度/回归/ANOVA + 三角验证矩阵 + 标准输出物。
- **可立即产出的二次综合**：8 条痛点假设（H1–H8）、三支柱价值/信任假设、5 条可证伪研究假设（H-A1~H-E1）、技术债→体验断点映射。

## 最重要的结论（假设态，待验证）
1. 三支柱中 **AI 测评是获客主引擎，区块链证书价值高度依赖雇主认可生态**，孤立上链不足以打动用户。
2. **"模拟上链"标注不会显著拉低价值感知，但虚假宣称"已上链"会踩合规雷区**（对齐 PRD 措辞红线）。
3. 当前**写死的学习路径**大概率在 usability 测试中被判"不准"，直接支撑 Real 自适应路径立项。
4. 在职用户（15–20 分/天）更吃"轻量 3D 虚拟面试"而非长视频课。

## 下一步
- 确认招募渠道与预算 → 启动 Phase 1 访谈。
- 可立即对 Lite 已跑通闭环做一轮 5 人可用性快测（Phase 1b），保 Lite 演示不踩雷。
- Real 立项前用 Phase 2 问卷量化三支柱优先级。
