## 1. 研究背景与业务对齐

### 1.1 产品三支柱与当前真相
- **A 智能测评**：登录 → 12 题测评 → DeepSeek 生成能力雷达图 + 报告 + 课程推荐（闭环已跑通；无 Key 时降级本地文案）。
- **B 元宇宙实训**：Three.js 静态 3D 场景（面试/课堂/会议/实训），AI 角色以发光光环 + 标签呈现，**非实时对话**（Real 阶段接 WebRTC）。
- **C 区块链证书**：`BlockchainService` 为**模拟**（假 txId，`verify` 恒返回 `true`），前端标注"模拟上链"。
- **课程/路径**：课程 CRUD 可用；学习路径为**写死文本**，非基于用户数据。

### 1.2 为什么现在必须做用户研究
1. **Lite 演示前**：需验证"学-练-证"闭环对评委/学员是否真讲得通，避免演示穿帮（尤指区块链"模拟"与元宇宙"静态"的措辞风险）。
2. **Real 立项依据**：区块链备案、元宇宙 AI 实时互动、自适应路径三大投入巨大，必须由用户价值证据排序，而非拍脑袋。
3. **避免假设驱动**：竞品分析已提出 6 类痛点（完课率塌方、同质化、证书价值弱、个性化不足、实训缺真实感、AI 信任脆弱），但均为二手推断，需一手验证其**严重度 × 普遍度**。
4. **已知技术债映射用户影响**：`B5` 报名/选课缺失、`B6` 生产无种子数据、`F-013` 证书模拟——这些不只是工程问题，更是用户体验断点。

## 2. 研究问题（Research Questions）

按"需求 → 痛点 → 价值 → 可用性 → B 端 → 包容性"六维设计，确保覆盖 C 端学员、B 端企业、以及平台自身可用性。

| 编号 | 研究问题 | 类型 | 主要方法 |
|------|----------|------|----------|
| **RQ1** | C 端学员在"学-练-证"闭环中的**核心任务目标、决策因素与流失节点**是什么？ | 探索性 | 深度访谈 + 旅程地图 |
| **RQ2** | 二手资料提出的 6 类痛点（完课率/同质化/证书价值/个性化/实训真实感/AI 信任）在真实用户中是否复现？**严重度与普遍度**如何？ | 验证性 | 访谈 + 问卷 |
| **RQ3** | AI 测评 / 元宇宙实训 / 区块链证书三支柱的**感知价值、信任度、付费意愿**如何排序？哪些场景真正打动用户？ | 验证性 + 优先级 | 访谈 + 问卷（ conjoint/排序） |
| **RQ4** | Lite 闭环（登录→测评→报告→课程→证书）的**任务完成率、效率、满意度**如何？3D 场景的眩晕/性能/可理解性问题？ | 评估性 | 可用性测试（think-aloud） |
| **RQ5** | B 端 HR/培训负责人对"可验证技能 + 沉浸式练习"的**采购决策因素、合规顾虑、预算区间**？ | 探索性 | B 端深度访谈 |
| **RQ6** | 不同年龄/设备/数字素养用户及**无障碍（视障/听障/运动障碍）需求**是否被满足？ | 包容性 | 可用性测试 + 无障碍审计 |

## 3. 总体方法论：混合方法设计（Mixed-Methods）

### 3.1 设计逻辑
采用 **探索性序列混合设计（Exploratory Sequential Mixed-Methods）**：
1. **先定性**（Phase 0 案头 + Phase 1 访谈）生成假设、探索未知维度；
2. **后定量**（Phase 2 问卷）在更大样本上验证普遍度、量化优先级、做细分差异；
3. **全程可用性测试**独立评估产品本身；
4. **三角验证**将三类数据交叉，提升结论可信度。

### 3.2 方法矩阵

| 阶段 | 方法 | 样本 | 回答的问题 | 周期 |
|------|------|------|-----------|------|
| Phase 0 | 二次数据分析（案头） | 现有文档 | RQ2 假设生成、基线 | 1 周（可立即） |
| Phase 1 | 半结构化深度访谈 | C 端 12–15 + B 端 6–8 | RQ1/RQ3/RQ5 | 2–3 周 |
| Phase 1b | 可用性测试（Lite 闭环 + 3D） | 学员 8–10 | RQ4/RQ6 | 1–2 周 |
| Phase 2 | 结构化问卷 | C 端 ≥200 + B 端 ≥30 | RQ2/RQ3 普遍度 | 2–3 周 |
| 持续 | 行为埋点（Real 阶段） | 全量 | RQ4 漏斗验证 | Real 上线后 |

## 4. 分阶段研究方案

### Phase 0 · 案头研究 / 二次数据分析（可立即启动）
- **输入**：竞品分析报告、项目认知报告、PRD、安全审计报告、现有测评/课程数据结构。
- **动作**：痛点归类、提取被引用但未验证的统计数据、生成"可证伪假设清单"、建立指标基线（如行业完课率 30–50% 作对比基准）。
- **产出**：第 8 节综合 + 研究假设清单（已成本文件交付物）。

### Phase 1 · 定性深度访谈
- **目标**：探索 RQ1/RQ3/RQ5，验证 RQ2 痛点假设。
- **提纲要点**（完整版见附录 A）：
  - 求职/转行/提升的近期经历与挫败；
  - 对"AI 给我打分""在虚拟环境练面试""证书上链"的直觉反应与信任；
  - 付费意愿与决策门槛；
  - B 端：采购流程、合规红线、ROI 计算方式。
- **时长**：45–60 分钟/人，录制 + 转录，知情同意。

### Phase 1b · 可用性测试（Lite 闭环 + 3D 场景）
- **任务卡**（见附录 B）：
  - T1 注册/登录 → 完成 12 题测评 → 解读雷达图；
  - T2 查看 AI 推荐课程 → 打开课程详情；
  - T3 查看"我的证书"并理解"模拟上链"标注；
  - T4 进入元宇宙静态场景并旋转视角；
  - T5（无障碍）仅键盘 / 屏幕阅读器完成 T1。
- **指标**：任务完成率、时长、错误数、SUS、情绪评分、眩晕/卡顿主观报告。

### Phase 2 · 定量问卷
- **核心量表**（见附录 C）：SUS（可用性）、NPS（推荐度）、TAM（技术接受度：感知有用/易用）、**AI 信任量表**（可解释性/公平性/兜底需求）、**支付意愿（WTP）**、痛点严重度 × 普遍度矩阵题。
- **样本量依据**：C 端 ≥200 满足因子分析（KMO≥0.6、样本/题项 ≥10）与细分对比（大学生/转行/在职三组 ANOVA）；B 端 ≥30 做描述统计与采购因素排序。
- **抽样**：分层配额（按身份、年龄段、城市线级），避免仅学生样本偏差。

## 5. 参与者标准与招募

### 5.1 C 端分层（对应竞品分析画像）
| 细分 | 特征 | 配额 | 招募渠道 |
|------|------|------|----------|
| 大学生/应届生 | 价格敏感、要写进简历 | 40% | 高校社群、实习群、校园合作 |
| 转行人群 25–35 | 要见效快+实战+督学 | 35% | 求职社区、转行训练营 |
| 在职员工 | 时间碎片（15–20 分/天）、与岗位挂钩 | 25% | 职场社群、企业内推 |

### 5.2 B 端
- 培训负责人 / HR / 业务线主管；行业覆盖制造、IT、零售、服务业；企业规模 50–2000 人（避开仅服务巨头的定位盲区）。

### 5.3 样本量统计依据
- **访谈饱和**：主题饱和（thematic saturation）通常 n=12–15/细分段即可覆盖绝大多数主题（Guest et al., 2006）。
- **可用性测试**：5 人可发现约 80% 可用性问题（Nielsen, 2000）；取 8–10 覆盖数字素养梯度与无障碍边界。
- **问卷**：≥30 为统计下限；≥200 支撑细分与信效度检验。

### 5.4 偏差防止（Bias Mitigation）
- 不只用学生样本（竞品分析提示在职占 70%）；
- 覆盖数字素养低→高梯度，含至少 2 名银发/低龄边缘用户；
- 访谈者不与被试有利益关联；避免引导式提问（如不问"你是不是觉得证书没用"，改问"你如何判断一张证书值不值得考"）；
- 招募时排除内部员工/亲友，避免同温层。

### 5.5 包容与无障碍
- 招募含视障（屏幕阅读器用户）、听障、运动障碍各至少 1 名；
- 测试任务提供键盘可达性与对比度检查；
- 3D 场景提供"减少动效"开关以降低眩晕风险。

## 6. 研究协议与工具

- **知情同意**：书面 + 口头，明确录制用途、数据留存期限（≤2 年）、可随时退出、匿名化承诺。
- **隐私合规**：测评/证书涉及个人信息，遵循最小化采集；AI 报告展示前脱敏；符合《个人信息保护法》。
- **工具栈**：
  - 录制：会议室录制 + 转录（如飞书/讯飞）；
  - 定性编码：Taguette / Dovetail / 表格亲和图；
  - 定量：问卷星/腾讯问卷 + SPSS/Python(pandas/statsmodels)；
  - 可用性：屏幕录制 + 眼动（可选）+ SUS 表单；
  - 无障碍审计：axe DevTools + 手动键盘/读屏走查。

## 7. 结果分析框架（核心方法论）

### 7.1 定性分析：主题分析六步法（Braun & Clarke）
1. **熟悉数据**：逐字转录、反复阅读；
2. **初始编码**：开放编码（见 7.5 示例）；
3. **寻找主题**：将编码聚为潜在主题；
4. **审查主题**：检查主题与编码的贴合度；
5. **定义命名**：明确每个主题的内涵与边界；
6. **撰写报告**：结合引语与业务含义。

**痛点矩阵**（输出物之一）：以"严重度（1–5）× 普遍度（出现访谈数/总样本）"二维定位，优先处理右上角（高严重×高普遍）。

### 7.2 定量分析
- **信度**：Cronbach's α ≥ 0.7（量表内部一致性）；重测信度（可选）。
- **效度**：内容效度（专家评审）+ 结构效度（探索性因子分析 EFA，KMO≥0.6、Bartlett 显著）。
- **描述统计**：均值/中位数/分布（SUS、NPS、WTP）。
- **相关/回归**：痛点严重度 → 流失意向、付费意愿 → 感知价值 的回归系数。
- **细分差异**：单因素 ANOVA / t 检验（大学生 vs 转行 vs 在职；数字素养高 vs 低）。
- **结构方程（可选）**：技术接受（TAM）与信任对使用意向的路径模型。

### 7.3 三角验证矩阵
| 研究问题 | 定性证据 | 定量证据 | 行为数据 | 一致性判定 |
|----------|----------|----------|----------|-----------|
| RQ2 痛点 | 访谈引语 | 问卷严重度均值 | 埋点放弃率 | 三方同向 → 高置信 |
| RQ3 价值排序 | 访谈优先级 | 排序题/WTP | 功能使用率 | 交叉确认 |

> 任一条结论若仅单一来源支持，标注"待验证"，不进入高优先级决策。

### 7.4 标准输出物
1. **实证用户画像**（≥3 个 C 端 + 1 个 B 端，含真实引语、目标、痛点、场景）；
2. **用户旅程地图**（学-练-证全链路，标注情绪曲线与断点）；
3. **可用性测试报告**（任务完成率/时长/错误/SUS/无障碍问题清单）；
4. **优先级矩阵**（痛点严重度×普遍度 + 三支柱价值排序）；
5. **产品建议清单**（高/中/低优先级，含预期影响与成功度量）。

### 7.5 编码示例（访谈 → 洞察）
```
原始引语：「我报过两三个课，都是前面看几天，后面就忘了，没人提醒我。」
开放式编码：#完课率塌方 #缺督学 #自主性不足
主题归集：[持久参与障碍] → 关联二手假设 H2（完课率）
业务含义：自适应路径 + 轻量督学（Real 阶段）直接命中该痛点；Lite 可在报告页加"学习提醒"兜底。
```

### 8.2 三支柱价值 / 信任假设（待量化排序）

- **A AI 测评**：获客主引擎（低成本、高增速），但**信任靠"人+透明"**——"筛查非判决""AI 初诊+导师复核"。→ 假设：学员接受度中高，但要求可解释与兜底（H6）。
- **B 元宇宙实训**：企业侧价值 >> 个人侧；个人端需 Web/手机轻量降门槛；聚焦"虚拟面试"高痛点场景。→ 假设：个人用户对"炫"敏感、对"用"更看重（H5/H7）。
- **C 区块链证书**：认知度高、信任"不可篡改"，但**实际求职价值高度依赖雇主生态**。→ 假设：证书价值 = 技术可信 × 雇主认可网络（H3/H8），孤立上链不足以打动用户。

### 8.3 可证伪研究假设清单（供 Phase 1/2 验证或推翻）

1. **H-A1**：学员愿为"AI 测评 + 个性化路径"付费的意愿显著高于为"区块链证书"付费。（预期被问卷 WTP 验证）
2. **H-B1**：在职用户（15–20 分/天）对"轻量 3D 虚拟面试"的完成意愿高于"长视频课"。（预期被任务偏好题验证）
3. **H-C1**：明确标注"模拟上链"不会显著降低学员对闭环的价值感知，但**虚假宣称"已上链"会**（合规风险）。（预期被表述对照实验验证）
4. **H-D1**：当前写死的推荐路径（非个性化）在可用性测试中会被多数用户感知为"不准/无关"。（预期被 T2 + 个性化题项验证 → 直接支持 Real 自适应路径立项）
5. **H-E1**：低数字素养用户在第 12 题测评流程中完成率显著低于高数字素养用户。（预期被 T1 分层验证 → 无障碍优化依据）

### 8.4 已知技术债 → 用户体验断点映射
| 技术项 | 用户可感知影响 | 研究动作 |
|--------|---------------|----------|
| B5 报名/选课缺失（仅 ElMessage 提示） | 学员"想学却报不了"的断点 | 可用性测试 T2 必现，记录受阻反应 |
| B6 生产无初始数据 | 首启空荡、无内容可看 | Lite 演示前需种子数据（见 PRD Open Q1） |
| F-013 证书模拟 | "模拟上链"标注是否被理解/是否被信任 | H-C1 对照实验 |

## 9. 伦理与合规

- **知情同意与隐私**：书面同意、数据最小化、匿名化、留存 ≤2 年、可撤回。
- **弱势群体**：低数字素养/无障碍用户不参与竞争性评价，仅观察协助。
- **AI/区块链合规提示**：研究涉及 AI 测评与（模拟）上链，需同步合规设计——个人信息保护（PIPL）、区块链备案（Real）、不向用户虚假宣称"已上链"（PRD §5 措辞红线）。
- **数据安全**：录制文件加密存储，访问授权到人。
